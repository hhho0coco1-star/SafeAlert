## 3. 페이지별 상세 분석

---

### 3-1. 랜딩 페이지 `/`

**목적:** 비로그인 사용자에게 서비스 소개, 회원가입 유도

**구성 섹션:**

| 섹션 | 내용 |
|------|------|
| Hero (좌) | 뱃지, 제목, 설명, 무료시작 버튼, 공공 API 출처 뱃지 |
| Hero (우) | 실시간 알림 피드 박스 (애니메이션 — 목업 데이터) |
| Stats | 5초 · 4종 · 99.9% 카운트업 애니메이션 |
| 핵심 기능 | 2×2 카드 (실시간 WebSocket, 지역별 구독, 카테고리 선택, 알림 이력) |
| CTA | 카테고리 뱃지 + 3단계 플로우(회원가입→지역설정→알림수신) + 버튼 |
| Footer | 로고, 소개, 이용약관/개인정보 |

**연결 API:**

| 동작 | API | 상태 |
|------|-----|------|
| 알림 피드 조회 | `GET /api/alerts/recent?limit=8` | ⬜ 미구현 → Mock Fallback |

**구현 포인트:**
- Hero 우측 피드: `GET /api/alerts/recent` 호출 → 실패 시 `MOCK_FEED_ALERTS` 상수로 fallback
- Stats (5초, 4종, 99.9%): 서비스 스펙 고정값 — API 불필요
- Stats 카운트업: `IntersectionObserver`로 화면 진입 시 시작
- 로그인 상태면 `/dashboard`로 리다이렉트

---

### 3-2. 로그인/회원가입 `/login`

**목적:** 이메일 로그인 + Google/Kakao OAuth2

**구성:**

| 탭 | 필드 | 특이사항 |
|----|------|----------|
| 로그인 | 이메일, 비밀번호, 비밀번호 찾기 | 비밀번호 표시/숨김 토글 |
| 회원가입 | 닉네임, 이메일, 비밀번호, 비밀번호 확인, 약관 동의 | 비밀번호 강도 인디케이터 3단계 |

**소셜 로그인:**
- Google 버튼 (Google 색상 SVG 아이콘)
- Kakao 버튼 (노란 배경 `#FEE500`)

**연결 API:**

| 동작 | API |
|------|-----|
| 이메일 로그인 | `POST /api/auth/login` |
| 회원가입 | `POST /api/auth/signup` |
| Google 로그인 | `GET /oauth2/authorization/google` |
| Kakao 로그인 | `GET /oauth2/authorization/kakao` |

**구현 포인트:**
- URL 쿼리 `?mode=signup`으로 탭 초기값 제어
- JWT 저장: `localStorage`에 `accessToken`, `refreshToken`
- 로그인 성공 → `/dashboard` 이동
- 이미 로그인 상태면 `/dashboard` 리다이렉트

---

### 3-3. 대시보드 `/dashboard`

**목적:** 구독 현황 요약 + 실시간 알림 수신

**구성:**

```
인사말 (이름, 구독 지역 수)
├── 미니 스탯 4개: 오늘 수신 알림 · 기상특보 · 지진 · 미세먼지
├── [실시간 알림 피드 카드]    [내 구독 현황 카드]
│    WebSocket 알림 목록        지역 수 · 지역명
│    (새 알림 위에서 추가)      카테고리 수 · 카테고리 뱃지
│    "알림 이력 전체 보기" →    마지막 수신 시각
│                               [구독 설정 변경 →]
└── 실시간 알림 모달 (WebSocket 수신 시 자동 팝업)
```

**연결 API:**

| 동작 | API |
|------|-----|
| 구독 현황 조회 | `GET /api/subscriptions` |
| 알림 이력 조회 (최근 7건) | `GET /api/notifications?size=7` |
| WebSocket 연결 | `WS /api/notifications/ws` (STOMP) |

**구현 포인트:**
- STOMP 구독: `/topic/user.{userId}` 채널 수신
- 새 알림 수신 → 피드 맨 위에 슬라이드인 애니메이션 + 모달 팝업
- 모달 색상 = 카테고리 색상과 동일
- 미니 스탯 4개는 API 데이터 기반 (오늘 날짜 필터링)

### 3-4. 구독 설정 `/subscriptions`

**목적:** 구독 지역 및 카테고리 관리

**구성:**

```
[지역 구독 설정 카드]
  시/도 드롭다운 → 시/군/구 드롭다운 → [+추가] 버튼
  등록된 지역 태그 (삭제 X 버튼 포함) | 0/5 카운터

[알림 카테고리 설정 카드]
  기상특보 [토글]
  지진     [토글]
  미세먼지 [토글]
  재난문자 [토글]

[설정 저장] 버튼
```

**연결 API:**

| 동작 | API |
|------|-----|
| 현재 구독 조회 | `GET /api/subscriptions` |
| 구독 생성 (최초) | `POST /api/subscriptions` |
| 지역 추가 | `POST /api/subscriptions/regions` |
| 지역 삭제 | `DELETE /api/subscriptions/regions/{regionCode}` |
| 카테고리 수정 | `PUT /api/subscriptions/categories` |
| 가능한 지역 목록 | `GET /api/subscriptions/regions/available` |

**구현 포인트:**
- 지역 최대 5개 제한 (클라이언트 + 서버 모두 검증)
- 시/도 선택 시 시/군/구 드롭다운 활성화
- 카테고리 1개 이상 선택 필수 (저장 전 검증)
- 저장 버튼 클릭 → 지역 변경(추가/삭제) + 카테고리 PUT 순서로 호출
- 성공 메시지 3초 후 자동 숨김

### 3-5. 알림 이력 `/history`

**목적:** 수신한 알림 목록 조회 및 필터링

**구성:**

```
[필터 카드]
  [전체] [기상특보] [지진] [미세먼지] [재난문자]  ← 카테고리 버튼
  [시작일] ~ [종료일]  [🔍 검색창]  [초기화]

[이력 리스트]
  ● 알림 제목                           [카테고리] [긴급도]  시각  >
    출처 · 지역
    ▼ 클릭 시 인라인 펼치기:
      상세 내용 (왼쪽 컬러 보더)
      발생시각 · 지역 · 출처

[페이지네이션]  < 1 2 3 >
```

**긴급도 뱃지:**

| 레벨 | 표시 | 색상 |
|------|------|------|
| critical | 긴급 | 빨간 배경 흰 글씨 |
| high | 높음 | 연빨간 배경 |
| medium | 보통 | 연노란 배경 |
| low | 낮음 | 회색 배경 |

**연결 API:**

| 동작 | API |
|------|-----|
| 알림 이력 조회 | `GET /api/notifications?category=&page=&size=7` |

**쿼리 파라미터:** `category`, `startDate`, `endDate`, `keyword`, `page`, `size`

**구현 포인트:**
- 페이지 크기: 7건
- 카테고리 버튼 클릭 → API 재호출
- 인라인 펼치기: 한 번에 하나만 열리도록 토글
- 날짜 필터: 브라우저 기본 `date input` 사용

### 3-6. 내 계정 `/profile`

**목적:** 계정 정보 조회 및 수정

**구성:**

```
[프로필 카드]
  아바타 (닉네임 첫 글자) | 닉네임, 이메일, Google 연동 뱃지, 가입일

[계정 정보 수정]
  닉네임 (수정 가능) | 이메일 (disabled)
  [저장] 버튼 → 인라인 성공/실패 메시지

[비밀번호 변경]
  소셜 로그인 시: 안내 메시지 + 버튼 비활성화
  이메일 로그인 시: 현재/새/확인 비밀번호 입력

[로그아웃]
  "현재 기기에서 로그아웃합니다. 실시간 알림 수신이 중단됩니다."
  [로그아웃] 버튼

[위험 영역]
  "모든 구독 정보와 알림 이력이 영구 삭제됩니다."
  [회원 탈퇴] 버튼 → 확인 모달 ('탈퇴합니다' 입력 후 활성화)
```

**연결 API:**

| 동작 | API |
|------|-----|
| 내 정보 조회 | `GET /api/auth/me` |
| 닉네임 수정 | `PUT /api/auth/me` |
| 로그아웃 | `POST /api/auth/logout` |
| 회원 탈퇴 | `DELETE /api/auth/me` |

**구현 포인트:**
- 소셜 로그인 여부: JWT 페이로드 또는 `/api/auth/me` 응답의 `oauthProvider` 필드 확인
- 소셜 로그인이면 비밀번호 변경 카드 비활성화
- 탈퇴 확인 입력값: `'탈퇴합니다'` 정확히 일치해야 버튼 활성화
- 로그아웃 → localStorage 토큰 삭제 → `/` 이동

### 3-7. 관리자 대시보드 `/admin`

**목적:** 시스템 통계 조회 + 수동 알림 발송

**접근 조건:** JWT의 `role = 'ADMIN'`

**구성:**

```
[통계 카드 4개]
  전체 회원 수 | 오늘 발송 수 | 활성 구독 수 | 시스템 상태

[최근 발송 알림]          [최근 가입 회원]
  실시간 피드 6건          회원 테이블 (이름/이메일/구독수/가입일/상태)
  (3초마다 자동 갱신)

[알림 수동 발송 폼]
  카테고리 선택 | 대상 지역 선택
  알림 제목 입력
  알림 내용 입력
  [발송하기] → 확인 모달 (요약 표시) → 확인 시 발송
```

**연결 API:**

| 동작 | API |
|------|-----|
| 통계 조회 | `GET /api/admin/stats/alerts` |
| 최근 알림 조회 | `GET /api/admin/alerts` |
| 수동 알림 발송 | `POST /api/admin/alerts/manual` |

**수동 발송 요청 Body:**
```json
{
  "category": "WEATHER",
  "regionCode": "11",
  "title": "알림 제목",
  "content": "알림 내용"
}
```

**구현 포인트:**
- `/admin` 접근 시 `role !== 'ADMIN'` → `/dashboard` 리다이렉트
- 발송 확인 모달에 입력 내용 요약 표시
- 발송 성공 → 폼 초기화 + 토스트 메시지

## 4. 상태 관리 설계

### 4-1. 전역 상태 (Context API 사용)

```
AuthContext
  ├── user: { userId, email, nickname, role, oauthProvider }
  ├── accessToken: string | null
  ├── isLoggedIn: boolean
  ├── login(accessToken, refreshToken) → localStorage 저장
  └── logout() → localStorage 삭제, WebSocket 해제

WebSocketContext
  ├── connected: boolean
  ├── subscribe(userId) → STOMP 연결
  ├── disconnect()
  └── onMessage(callback) → 알림 수신 핸들러
```

### 4-2. JWT 저장 및 갱신

```
localStorage
  ├── accessToken   (15분 만료)
  └── refreshToken  (7일 만료)

Axios 인터셉터
  ├── 요청 시: Authorization: Bearer {accessToken} 헤더 자동 추가
  └── 401 응답 시: POST /api/auth/refresh → 재발급 → 원래 요청 재시도
```

### 4-3. Protected Route

```jsx
// 로그인 필요 페이지
<ProtectedRoute> → 비로그인 시 /login 리다이렉트

// 관리자 전용 페이지
<AdminRoute> → role !== 'ADMIN' 시 /dashboard 리다이렉트

// 비로그인 전용 (로그인 페이지)
<GuestRoute> → 로그인 상태 시 /dashboard 리다이렉트
```

## 7. 주의사항 및 결정 사항

| 항목 | 결정 |
|------|------|
| CSS 방식 | Tailwind CSS (목업 스타일 최대한 재현) |
| 아이콘 | Tabler Icons (목업과 동일, `@tabler/icons-react`) |
| 상태 관리 | Context API (Redux 미사용 — 규모 적절) |
| JWT 저장소 | localStorage (XSS 위험 존재, 포트폴리오 수준) |

## 8. API 연동 전략

### 8-1. Mock Fallback 패턴

notification-service가 구현되기 전까지 아래 패턴으로 처리한다.

```js
// 사용 예시 (랜딩 피드)
try {
  const { data } = await api.get('/api/alerts/recent?limit=8')
  setAlerts(data.data)
} catch {
  setAlerts(MOCK_FEED_ALERTS)   // notification-service 완성 후 이 줄 제거
}
```

- API 성공(200): 실제 데이터 표시
- API 실패(네트워크 오류 / 404 / 서비스 미구현): MOCK 상수로 fallback
- notification-service 구현 완료 후 catch 블록의 fallback 코드 제거
