# SafeAlert 개발 환경

> 행동 규칙은 `CLAUDE.md` 참조. 이 문서는 환경·접속·실행 정보만 담는다.

## 1. 개발 환경 구조 (기본)
코드 수정마다 Docker 재빌드하지 않는다. 아래 구조를 기본으로 사용한다.

- **인프라**: `docker compose up -d postgresql redis kafka mongodb`
- **백엔드**: 각 서비스 직접 실행 (Spring DevTools가 코드 저장 시 자동 재시작)
- **프론트엔드**: `cd frontend && npm run dev` → http://localhost:5173

> Docker 재빌드 / K8s 재배포는 최종 배포 검증 목적으로만 사용한다.

## 2. 백엔드 서비스 실행 명령어 / 포트

| 서비스 | 실행 명령어 | 포트 |
|--------|------------|------|
| api-gateway | `./gradlew bootRun` | 8080 |
| auth-service | `./gradlew bootRun --args='--spring.profiles.active=local'` | 8081 |
| notification-service | `./gradlew bootRun` | 8083 |
| subscription-service | `./gradlew bootRun` | 8085 |
| alert-collector-service | `./gradlew bootRun` | 8086 |
| alert-processor-service | `./gradlew bootRun` | 8087 |

> auth-service는 OAuth2 시크릿 때문에 `local` 프로파일이 필요하다.
