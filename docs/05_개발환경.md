# SafeAlert 개발 환경

> 행동 규칙은 `CLAUDE.md` 참조. 이 문서는 환경·접속·실행 정보만 담는다.

## 1. 개발 환경 구조 (기본)
코드 수정마다 Docker 재빌드하지 않는다. 아래 구조를 기본으로 사용한다.

- **인프라**: `docker compose up -d postgresql redis kafka mongodb`
- **백엔드**: 각 서비스 직접 실행 (Spring DevTools가 코드 저장 시 자동 재시작)
- **프론트엔드**: `cd frontend && npm run dev` → http://localhost:5173

> Docker 재빌드 / K8s 재배포는 최종 배포 검증 목적으로만 사용한다.

## 2. 백엔드 서비스 실행 명령어 / 포트

| 서비스 | 실행 명령어 | 포트 |
|--------|------------|------|
| api-gateway | `./gradlew bootRun` | 8080 |
| auth-service | `./gradlew bootRun --args='--spring.profiles.active=local'` | 8081 |
| notification-service | `./gradlew bootRun` | 8083 |
| subscription-service | `./gradlew bootRun` | 8085 |
| alert-collector-service | `./gradlew bootRun` | 8086 |
| alert-processor-service | `./gradlew bootRun` | 8087 |

> auth-service는 OAuth2 시크릿 때문에 `local` 프로파일이 필요하다.

## 4. 관측 가능성(Observability) 대시보드

> `/run` 실행 후 아래 주소에서 모니터링 가능. 외부 사이트 접속 불필요.

| 도구 | 주소 | 용도 |
|------|------|------|
| **Prometheus** | http://localhost:9090 | 메트릭 원본 데이터 조회 (PromQL 쿼리) |
| **Grafana** | http://localhost:3000 | JVM 힙·스레드·Kafka Lag 그래프 대시보드 (admin/admin) |
| **Jaeger** | http://localhost:16686 | API 요청 분산 트레이싱 — 어느 서비스를 거쳤는지, 각 단계 소요시간 확인 |
| **Kibana** | http://localhost:5601 | 6개 서비스 로그 통합 검색 (Discover → safealert-* 인덱스) |

### 각 도구 한 줄 요약

- **Prometheus**: "지금 JVM 힙이 몇 MB야?" 같은 숫자 데이터 저장소
- **Grafana**: Prometheus 데이터를 그래프로 시각화. JVM 대시보드(ID: 4701) 임포트됨
- **Jaeger**: 로그인 API 요청이 api-gateway → auth-service → Redis 순으로 몇 ms 걸렸는지 추적
- **Kibana**: 모든 서비스 로그를 한 화면에서 검색. `service_name: auth-service` 같이 필터링 가능

## 5. K8s 모니터링 스택 (safealert-monitor 네임스페이스)

> 모든 모니터링 도구가 K8s `safealert-monitor` 네임스페이스에 Helm/YAML로 배포되어 있다.
> 아래 포트포워딩 명령어로 로컬에서 접근 가능하다.

| 도구 | 포트포워딩 명령어 | 로컬 접속 주소 |
|------|----------------|--------------|
| **Prometheus** | `kubectl port-forward svc/prometheus-server 9090:80 -n safealert-monitor` | http://localhost:9090 |
| **Grafana** | `kubectl port-forward svc/grafana 3000:80 -n safealert-monitor` | http://localhost:3000 (admin/admin) |
| **Jaeger** | `kubectl port-forward svc/jaeger-query 16686:16686 -n safealert-monitor` | http://localhost:16686 |
| **Kibana** | `kubectl port-forward svc/kibana 5601:5601 -n safealert-monitor` | http://localhost:5601 |

### K8s 내부 서비스 주소 (서비스 간 통신용)

## 6. 트러블슈팅

### 서버 실행 관련

**Q. `./gradlew bootRun` 실행 시 포트 이미 사용 중 오류**
```
Web server failed to start. Port 8081 was already in use.
```
→ 이미 실행 중인 서비스를 종료 후 재시작
```powershell
# 해당 포트를 사용하는 프로세스 종료 (예: 8081)
$proc = Get-NetTCPConnection -LocalPort 8081 | Select-Object -ExpandProperty OwningProcess
Stop-Process -Id $proc -Force
```

---

**Q. auth-service 실행 시 OAuth2 관련 오류**
```
Could not resolve placeholder 'GOOGLE_CLIENT_ID'
```
→ `--spring.profiles.active=local` 옵션 없이 실행했기 때문
```bash
./gradlew bootRun --args='--spring.profiles.active=local'
```

---

**Q. Kafka 연결 실패 오류**
```
org.apache.kafka.common.errors.TimeoutException: Topic not present
```
→ Docker Compose 인프라가 실행되지 않은 상태. 먼저 인프라 실행
```bash
docker compose up -d postgresql redis kafka mongodb
```
Kafka 기동까지 약 30초 소요. `healthcheck`가 통과될 때까지 대기.

---

**Q. subscription-service 시작 시 DB 연결 오류**
```
Connection to localhost:5432 refused
```
→ PostgreSQL 컨테이너가 아직 초기화 중. 30초 후 재시도

---

**Q. 프론트엔드에서 API 요청 시 CORS 오류**
→ API Gateway가 실행되지 않은 상태. api-gateway 먼저 실행 확인
```bash
curl http://localhost:8080/actuator/health  # UP이어야 함
```

---

### Docker 관련

**Q. `docker compose up` 시 orphan containers 경고**
```
Found orphan containers ([safealert-xxx-1])
```
→ 무시해도 되거나, `--remove-orphans` 옵션으로 정리
```bash
docker compose up -d postgresql redis kafka mongodb --remove-orphans
```

---

**Q. Elasticsearch 컨테이너가 계속 재시작됨**
→ 메모리 부족. `vm.max_map_count` 설정 필요 (Linux/WSL 환경)
```bash
sudo sysctl -w vm.max_map_count=262144
```

---

### K8s 관련

**Q. minikube 파드가 Pending 상태로 멈춤**
```bash
kubectl describe pod <pod-name> -n safealert-app
# Events 섹션에서 원인 확인
```
→ 주로 메모리 부족. minikube 재시작 시 `--memory=4096` 옵션 필요

**Q. 포트포워딩 연결이 끊김**
→ 포트포워드는 터미널 세션이 살아있어야 유지됨. 새 터미널에서 재실행
```bash
kubectl port-forward svc/grafana 3000:80 -n safealert-monitor
```
