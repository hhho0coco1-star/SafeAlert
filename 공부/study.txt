★ 용어 정리 ★

1. Minikube : 내 개인 PC에서도 쿠버네티스를 연습해볼 수 있게 해주는 '로컬용 연습장'

2. Kubernetes : 여러 대의 서버에 흩어진 컨테이너를 자동으로 관리·운영해주는 '컨테이너 지휘자'

3. Kafka : 대용량 데이터를 끊김 없이 실시간으로 전달하고 안전하게 보관하는 '데이터 고속도로'

4. Redis : 데이터를 메모리에 저장하여 압도적인 속도를 자랑하는 '초고속 데이터 저장소 및 캐시 시스템'

5. Spring -> @Bean : 개발자가 직접 생성한 객체를 Spring 컨테이너가 관리하는 
공용 객체로 등록하기 위해 메서드 위에 붙이는 표식

6. Spring -> @Component : Spring 컨테이너라는 거대한 공장에 
"이 클래스를 조립 부품으로 등록해줘"라고 요청하는 이름표

7. Spring -> @Configuration : Spring 컨테이너에게 객체들을 어떻게 만들고 서로 
어떻게 연결할지 알려주는 '공식 설계도'

8. 의존성 -> 의존성이 줄어든다는 것은 코드 간의 '결합도'가 낮아진다는 뜻이며, 
이는 곧 "수정하기 쉽고, 관리하기 편하며, 튼튼한 코드"가 된다는 것을 의미

9. @Transactional :  여러 DB 작업을 하나의 세트로 묶어서, 
도중에 하나라도 실패하면 모든 작업을 없던 일로 되돌려주는 안전장치

10. @SpringBootApplication : "애플리케이션 설정을 자동으로 하고, 
필요한 부품(Bean)들을 알아서 찾아 등록한 뒤, 서버를 실행해라"

11. deployment.yaml : "앱 실행 설정"

12. service.yaml : "앱에 접근할 수 있는 주소 설정"

13. @Slf4j : 로그(Log)를 아주 간편하게 남길 수 있게 해주는 도구

14. MCP(Model Context Protocol) : AI와 데이터 소스를 연결하는 '소프트웨어계의 USB 표준'

◇ Prompt : 좋은 프롬프트를 만드는 것은 AI에게 단순한 '질문'을 던지는 것이 아니라, 
내가 원하는 결과물이 나올 수 있도록 '명확한 업무 지시서'를 작성하는 과정과 같다.

- 역할 부여하기 (Assign a Persona)
AI에게 특정 분야의 전문가라는 정체성을 부여하세요. 답변의 톤과 전문성 수준이 확연히 달라집니다.
예시: "너는 10년 차 시니어 풀스택 개발자야.", "너는 실무 중심의 취업 컨설턴트야."

- 구체적인 배경과 상황 설명 (Context)
AI는 사용자가 처한 상황을 알지 못합니다. 무엇을 위해 이 결과물이 필요한지 맥락을 제공해야 합니다.

안 좋은 예: "이직 사유 좀 써줘."

좋은 예: "공백기가 1년 정도 있는데, 이 기간에 프로젝트를 하며 기술 역량을 쌓았다는 점을 강조해서 
긍정적인 이직 사유 문장을 만들어줘."

- 명확한 작업 지시 (Task & Instruction)
수행해야 할 행동을 명확한 동사로 표현하세요.

핵심: 요약해줘, 분석해줘, 코드를 작성해줘, 리팩토링해줘 등.

- 제약 사항 및 형식 지정 (Constraints & Format)
답변의 길이, 말투, 포함될 내용, 결과물의 형태를 미리 정해주면 수정 시간을 크게 줄일 수 있습니다.

형식: 표(Table) 형태, JSON 데이터, 글머리 기호(Bullet points) 등.

제약: "전문 용어는 지양하고 쉬운 비유를 들어줘.", "300자 이내로 작성해줘."

- 예시 제공 (Few-Shot Prompting)
원하는 결과물의 샘플을 한두 개 보여주면 AI는 그 패턴을 완벽하게 학습합니다.

예시: "질문: 사과 -> 답변: 과일 / 질문: 사자 -> 답변: 동물 / 질문: 자바스크립트 -> 답변: [이곳을 채워줘]"

★ React + Vite를 사용하는 이유 ★

[ React ]
- 화면을 '컴포넌트'라는 작은 조각으로 나눠서 만드는 UI 라이브러리
- 예를 들어 Navbar, 알림카드, 버튼 등을 각각 독립된 파일로 만들고 조립하는 방식
- 상태(State)가 바뀌면 해당 부분만 자동으로 다시 그려줘서 빠르고 효율적
- 전 세계에서 가장 많이 쓰는 프론트엔드 라이브러리 → 취업 시장에서 수요가 가장 높음

[ Vite ]
- React 프로젝트를 실행하고 빌드해주는 도구 (Create React App의 후계자)
- 기존 CRA(Create React App)와 비교했을 때:
  · 개발 서버 시작 속도: CRA는 10~30초 → Vite는 1~2초
  · 코드 수정 후 반영 속도: CRA는 느림 → Vite는 거의 즉시 반영
- 빠른 이유: 브라우저가 이해하는 방식(ESM)으로 직접 파일을 제공해서 번들링 과정을 생략

[ 한 줄 요약 ]
React = "화면을 어떻게 만들지" 담당
Vite  = "그 React 프로젝트를 빠르게 실행/빌드" 담당

둘을 함께 쓰면 → 빠른 개발 환경 + 실제 배포 가능한 결과물을 동시에 얻을 수 있다.
